    <article class="markdown-body entry-content" itemprop="text"><h1><a id="user-content-recastly-redux" class="anchor" aria-hidden="true" href="#recastly-redux"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Recast.ly Redux</h1>
<p>From our previous sprint, we now have a simple yet functional React application hooked up to live YouTube data.  This is spectacular, and now it's time to consider adding features to our application.  But as we plan, more features turn into more components, handlers must be passed further up and down the component tree, or sometimes from laterally across our app... yikes!  This is getting prohibitively complex very quickly!</p>
<p>One popular solution to the problem of increased complexity is flux design.  This paradigm stresses a <em>single state object</em> as the sole "source of truth" for your application, and defines all user <em>actions</em> as changes to that state. Actions themselves are simply objects (or, if we are using Redux Thunk, functions), which must pass through a <em>reducer function</em> which defines how each action alters the state.  In this sprint, we will walk through the process of converting our codebase to use Redux, a flux implementation that works very well with React.</p>
<p>We have collected some useful <a href="#resources">resources</a>, which you are encouraged to consult.</p>
<h2><a id="user-content-high-level-goals-of-this-sprint" class="anchor" aria-hidden="true" href="#high-level-goals-of-this-sprint"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>High Level Goals of this Sprint</h2>
<ul>
<li>Learn about Redux and the principals of flux architecture.</li>
<li>Gain exposure to front-end code bundling using Webpack.</li>
<li>Get some experience using thunks to dispatch asynchronous actions.</li>
</ul>
<h2><a id="user-content-project-set-up" class="anchor" aria-hidden="true" href="#project-set-up"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Project set up</h2>
<h3><a id="user-content-webpack" class="anchor" aria-hidden="true" href="#webpack"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Webpack</h3>
<p>Webpack is the official bundler for React applications.  All React code should be bundled into a single file before being served up to the client.  This achieves two aims:</p>
<ul>
<li>As our codebase grows, it becomes increasingly cumbersome to manually add script tags to all of our source files to the index.html page.  Bundling allows us to only add a single script tag, which references the bundled file.</li>
<li>Multiple script tags means multiple requests, and this means more loading time.  Serving up one big file rather than dozens of smaller files reduces multiple request cycles.
In this sprint, we've given you a pre-built Webpack file.  You'll have to install Webpack (using <code>npm install</code> to install from <code>package.json</code>). Since locally installing webpack saves executables into <code>node_modules/.bin</code>, you'll need to run webpack using the npm script we provided for you, <code>npm run build</code> (see <a href="https://docs.npmjs.com/misc/scripts#path" rel="nofollow">npm docs</a> for more info). Some useful flags to consider adding to this invocation:</li>
<li><code>-d</code>: runs Webpack in debug mode, which provides useful error messages to help you understand why Webpack isn't able to bundle your files.</li>
<li><code>-w</code>: this is the 'watch' flag, and sets webpack to rerun its bundling process every time it detects changes in the source files.</li>
</ul>
<p>You will experience Webpack more in-depth later on in your Hack-Reactor life, but for now, this is as involved as we need to get.</p>

<p>Redux Thunk is a library that allows us to dispatch actions asynchronously.  When Thunk is registered as middleware, our actions are able to return not just objects but functions which can do more advanced things (like invoke <code>dispatch</code> themselves!)</p>

<li class="task-list-item"><input type="checkbox" id="" disabled="" class="task-list-item-checkbox"> You'll have to use the <code>applyMiddleware</code> method from Redux to allow your store to interpret your thunks. <li class="task-list-item"><input type="checkbox" id="" disabled="" class="task-list-item-checkbox"> in <code>actions/search.js</code>, write out a thunk to interact with the YouTube API.  Remember that instead of returning an object, we will now be able to return a function!</li>
</ul>
